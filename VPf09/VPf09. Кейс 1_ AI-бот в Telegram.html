Результат дня

Реализуем подключение LLM-модели (например, GPT-5 mini) к интерфейсу Telegram-бота для создания интеллектуального ассистента. Разберем, как работать с API OpenAI и BotFather, настроим «короткую память» диалога для сохранения контекста и научимся управлять ролями бота через системные промпты. Расширим возможности агента, добавив внешние функции (например, генерацию видео или изображений) и логирование стоимости запросов.

Теория на сегодня

1. Подключение LLM к Telegram-боту

В ближайших трёх уроках мы подытожим всё изученное касательно промптов, памяти, дополнительных инструментов, и так далее. В этом уроке мы будем подключать интерфейс в виде Telegram-бота к LLM-модели, чтобы мы могли с ней взаимодействовать.

Что такое LLM? ↘️

LLM чаще всего используются как:

  • ИИ-ассистенты (агенты);
  • Чаты поддержки (обработка стандартных пользовательских запросов, не требующих оператора);
  • Генераторы текстов.

LLM имеют два вида памяти:

  • “Короткая” — память, с которой мы взаимодействуем в плане истории диалога. В ней содержится системный промпт, запросы и ответы модели (контекст диалога).
  • “Долгая” — данные, хранящиеся в базе данных, и прочая информация, которая не зависит от того, кто именно производит запрос.

 

2. Практический кейс. Создание чат-бота Telegram

Подключение к API ↘️

Давайте ещё раз проговорим, где нам нужно брать данные для подключения Telegram-бота.

  1. Токен для бота. В Telegram открываем диалог с BotFather — сервисом по созданию ботов. Можно пользоваться либо командами в диалоге, либо нажать кнопку Open и инициализировать приложение.
  1. Нажимаем на Create New Bot.
  1. Зададим боту произвольное название (например, zerotest1211) и свободное имя пользователя. Обязательно, чтобы в конце имени пользователя был постфикс bot. Можно также оставить произвольное описание. Жмём на Create Bot.
  1. У нас появится уведомление о генерации API токена. Нажимаем Copy для его копирования. В любой момент можно нажать на Revoke для удаления данного токена.
  1. Вставляем этот токен в файл .env нашего проекта, в переменную BOT_TOKEN.
  1. Теперь получим токен OpenAI. Переходим на сайт платформы OpenAI. Заходим в аккаунт (можно через почту или аккаунт Google). Переходим в Settings → Billing. Убедись, что у тебя на счету есть некоторая сумма для осуществления запросов. Когда баланс достигнет нуля, наши запросы перестанут работать.
  1. Заходим в раздел API keys. Нажимаем Create new secret key. Вводим имя и выбираем тестовый проект. Нажимаем на кнопку Create secret key.
  1. Скопируй токен. Будь внимателен, этот диалог появится один раз, в дальнейшем скопировать токен будет нельзя, и придётся создавать новый.
  1. Вставь его в файл .env выше в качестве переменной OPENAI_API_KEY или аналогичной.
  1. Заодно поменяем используемую модель. Перейди в Default project → Models. Например, возьмём модель GPT-5 mini. Копируем тег модели, нажав на кнопку правее её названия, и вставляем в файл .env как переменную среды OPENAI_MODEL.

 

3. Запуск бота Telegram и клиента OpenAI

Запуск бота Telegram и клиента OpenAI ↘️

После того, как мы получили необходимые токены, мы переходим в Cursor. Первое, что нам необходимо — это написать промпт. В нём мы должны указать, что мы делаем бота на основе aiogram и OpenAI API. Явно потребуем, чтобы бот обладал контекстной памятью, хранил системные промпты и последние сообщения пользователя, и имел несколько ролей. Ответ также должен содержать комментарии и разделы, для облегчения расширения.

В результате мы, как и задавалось, сгенерировали файлы config.py с конфигурацией бота, memory.py с управлением памятью диалогов, добавлением сообщений, взаимодействием с файлами и получением истории, и main.py с логикой работы сервера (инициализация, обработчики команд пользователя и запросов Telegram и OpenAI). Логика работы в данном случае асинхронная, для возможности быстрого ответа сразу нескольким пользователям.
Также есть файлы prompts.json с нашими заданными промптами ролей и requirements.txt с требуемыми библиотеками. В качестве зависимостей мы задаём aiogram, OpenAI и python-dotenv для загрузки переменных среды.

Запускаем наше приложение. Для этого в терминале:

  1. Вызываем виртуальное окружение python командой py -m venv venv.
  1. Активируем виртуальное окружение командой .\venv\Scripts\activate .
  1. Устанавливаем зависимости под виртуальным окружением командой pip install -r .\requirements.txt. Ждём установки зависимостей.
  1. Запускаем бота python .\main.py.

Переходим в Telegram. Инициализируем бота, прописав в диалоге команду /start. Командой /mode мы можем менять режим (роль) агента. В дальнейшем, в рамках расширения, мы можем добавлять больше ролей для уточнения ответа.

Наш помощник может отвечать на общие вопросы и запоминает контекст диалога.

 

4. Добавление функций внутрь ИИ-агента

Добавление функций внутрь ИИ-агента ↘️

Давайте добавим функцию, которая бы позволила нам генерировать видео. Заодно попросим LLM организовать более подробное логирование и обработчик стоимости запросов.

После каждой команды в терминале отображаются логи с нашими действиями и стоимость запроса (в USD). Отдельно осуществляется логирование ошибок.

У нас появилась отдельная команда /video. Сейчас мы реализовали бота таким образом, что для генерации видео требуется переход в режим генерации видео через /mode.

Убедись в том, что нейросеть правильно определяет функцию генерации видео. В соответствии с референсом на сайте платформы OpenAI она должна называться openai.video.create(). Функция принимает на вход JSON объект с указанием модели и промпта. Другие параметры на вход не передаются. 

<code class="language-JavaScript" style="white-space:pre-wrap;word-break:break-all">import OpenAI from 'openai';
const openai = new OpenAI();
let video = await openai.videos.create({
    model: 'sora-2',
    prompt: "A video of the words 'Thank you' in sparkling letters",
});
console.log('Video generation started: ', video);</code>

Вместо этого мы также можем сделать генерацию изображений, которая уже не будет требовать авторизации. К сожалению, модели OpenAI вроде gpt-image-1-mini также требую авторизацию. Поэтому, генерацию изображений реализуем на базе GenAPI. Пишем ИИ-агенту Cursor, что мы хотим сделать функцию генерацию фото с использованием данной модели. Вновь обращай особое внимание на то, какие параметры передаются на вход модели, и поддерживаются ли они ей.

В результате мы получаем масштабируемого бота, которого можно доработать другими функциями, большим количеством ролей и моделей, и так далее. LLM при этом будет являться ядром нашей системы.
